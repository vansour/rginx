# v0.1.3-rc.2

## Summary

- Complete the staged TLS/SSL roadmap across config modeling, downstream TLS policy, downstream mTLS, upstream TLS, runtime observability, and migration tooling.
- Add Debian packaging, GitHub Release asset expansion, and signed APT repository publishing for stable releases.
- Strengthen admin and `check` diagnostics with certificate fingerprints, chain analysis, TLS runtime views, and clearer reload boundaries.

## Highlights

- TLS config boundaries are now explicit: listener/server TLS policy stays at the listener layer, while vhost TLS is modeled as certificate-only override material.
- Downstream TLS now supports stronger policy control including cipher suite selection, key exchange group selection, session resumption, and session ticket toggles.
- Downstream mTLS now exposes client certificate identity in request context, access logs, counters, and admin status/snapshot output.
- Upstream TLS now has explicit `server_name` behavior to align with `proxy_ssl_server_name`, while `server_name_override` continues to cover `proxy_ssl_name` semantics.
- TLS runtime observability now includes listener-level TLS status, certificate expiry tracking, SHA-256 fingerprints, chain diagnostics, SNI/default-certificate bindings, and richer `check` output.
- `migrate-nginx` now recognizes common SSL and `proxy_ssl_*` directives and migrates the supported subset into RON with warnings for manual follow-up where needed.
- Packaging and release tooling now produce `.deb` artifacts, include APT repository publishing scripts, and expand release automation for stable tags.

## Validation

- `cargo test --workspace --locked`
- `cargo test -p rginx --test migrate --test tls_policy --test downstream_mtls --test upstream_mtls --test upstream_http2 --test upstream_server_name --test access_log --test admin --test check --test reload`
- `cargo run -p rginx -- --version`

## Release Notes Draft

`v0.1.3-rc.2` is the first release candidate that closes the current TLS roadmap end-to-end. It upgrades TLS configuration boundaries, downstream and upstream TLS behavior, mTLS observability, runtime diagnostics, nginx SSL migration coverage, and packaging/release infrastructure in one pass.

Focus areas for rc validation are certificate diagnostics, SNI/default-certificate behavior, upstream TLS name handling, admin/check visibility, and the Debian/APT release pipeline.
